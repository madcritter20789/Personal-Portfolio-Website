Mechanic AI

Unity game mechanics and AI systems in action

Frame Recorder

Unity Tool  |  Gameplay Capture

A frame-based replay system in Unity that records and restores game state per-frame using a modular architecture where individual components (transforms, physics, animations) implement a shared Recorder interface. Frame states are stored in dictionaries keyed by frame number, enabling efficient lookup, smooth scrubbing, and flexible timeline changes. A central PlaybackController manages recording and playback modes while physics is disabled during playback for accurate state restoration — powering rewinding, killcams, instant replays, and debugging tools without altering the core system.

Procedural Generation

Unity Algorithm  |  Level Design

Procedural terrain generation using layered Perlin noise to create realistic landscapes from a height map, where multiple noise octaves add large-scale shapes and fine surface detail. Height data is shaped with fall-off maps to form islands or bounded terrain, then converted into a 3D mesh with vertices, triangles, and UVs. Visual variety comes through height-based biome coloring (water, grass, snow) rendered efficiently via vertex colors — producing vast, varied worlds with minimal manual design.

Motion Matching

Unity Animation  |  Character AI

A data-driven approach where every frame of animation is treated as a searchable pose in a database. Instead of manually defining transitions, the system continuously selects the best-fitting pose by comparing the character's current state and predicted trajectory against precomputed animation data via a cost function. Key features like foot positions, velocities, and future trajectory guide selection, while thresholds, clip-change penalties, and limited search frequency ensure stability. Built on Unity's Playables API — making new animations as simple as baking them into the database for scalable, responsive, natural character movement.
